# Unity Editor

> **Difficulty:** Beginner  
> **Estimated Reading Time:** 8 minutes  
> **Prerequisites:** None

---

# Overview

The Unity Editor is the primary development environment used to create games and interactive applications.

It provides tools for designing scenes, managing assets, writing scripts, and testing gameplay.

---

# Purpose

The Unity Editor allows developers to:

- Create scenes
- Place GameObjects
- Manage assets
- Write C# scripts
- Test gameplay
- Build projects for multiple platforms

---

# Main Windows

## Scene View

The Scene View is used to build and edit the game world.

Developers can:

- Move objects
- Rotate objects
- Scale objects
- Navigate the scene

---

## Game View

The Game View displays what the player will see during gameplay.

It is used for:

- Testing gameplay
- Previewing UI
- Checking cameras

---

## Hierarchy

The Hierarchy window displays every GameObject in the current scene.

Examples:

- Player
- Camera
- Lights
- Environment
- UI

---

## Inspector

The Inspector displays the properties of the selected GameObject.

Examples:

- Transform
- Components
- Materials
- Scripts

---

## Project Window

The Project window stores every asset in the project.

Examples:

- Prefabs
- Scripts
- Materials
- Textures
- Audio
- Animations

---

## Console

The Console displays:

- Errors
- Warnings
- Debug messages

Developers commonly use Debug.Log() while testing.

---

# Common Tools

| Tool | Purpose |
|------|---------|
| Move | Move GameObjects |
| Rotate | Rotate GameObjects |
| Scale | Resize GameObjects |
| Rect Tool | Edit UI |
| Transform Tool | Combined manipulation |

---

# Best Practices

✔ Keep the Hierarchy organized.

✔ Create folders inside the Project window.

✔ Name GameObjects clearly.

✔ Frequently save your scenes.

---

# Common Mistakes

❌ Using default object names.

❌ Storing assets in the root folder.

❌ Ignoring Console errors.

❌ Creating disorganized scenes.
