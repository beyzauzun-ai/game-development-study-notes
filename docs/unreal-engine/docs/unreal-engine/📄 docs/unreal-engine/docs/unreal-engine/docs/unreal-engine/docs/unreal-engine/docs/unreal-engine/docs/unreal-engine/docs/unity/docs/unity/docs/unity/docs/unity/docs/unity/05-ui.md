# Overview

Unity's User Interface (UI) system enables developers to create interactive menus, HUDs, inventory screens, settings panels, and other visual elements.

The UI system is built around the **Canvas** component, which acts as the root container for all interface elements.

- Create common UI elements
- Organize interfaces efficiently
- Build responsive layouts

# Canvas

The **Canvas** is the foundation of every Unity UI.

Without a Canvas, UI elements cannot be displayed.

Common Canvas Render Modes:

- Screen Space - Overlay
- Screen Space - Camera
- World Space

---

# Common UI Elements

## Text (TextMeshPro)

Displays textual information.

Examples:

- Player Score
- Timer
- Objectives
- Dialog

> **Note**
>
> Unity recommends **TextMeshPro (TMP)** instead of the legacy Text component for better rendering quality and flexibility.

---

## Button

Buttons trigger actions when clicked.

Examples:

- Start Game
- Continue
- Save
- Exit

Typical Events:

- OnClick()

---

## Image

Displays:

- Icons
- Backgrounds
- Logos
- Character Portraits

---

## Slider

Allows users to adjust values.

Examples:

- Music Volume
- Brightness
- Mouse Sensitivity

---

## Toggle

Represents an On/Off option.

Examples:

- Fullscreen
- VSync
- Enable Music

---

## Input Field

Allows users to enter text.

Examples:

- Player Name
- Chat
- Login

---

# Event System

The **Event System** manages player interaction with UI elements.

It detects:

- Mouse clicks
- Keyboard input
- Touch input
- Gamepad navigation

---

# UI Layout Components

Unity provides automatic layout tools.

Common Layout Components:

- Vertical Layout Group
- Horizontal Layout Group
- Grid Layout Group
- Content Size Fitter

These components help create flexible interfaces.

---

# Responsive UI

A responsive interface adapts to different screen sizes.

Recommendations:

✔ Use Anchors

✔ Use Layout Groups

✔ Test multiple resolutions

✔ Avoid absolute positioning

---

# HUD (Heads-Up Display)

A HUD displays gameplay information while the game is running.

Typical HUD Elements:

- Health
- Ammo
- Mini Map
- Compass
- Quest Tracker
- Crosshair

---

# Best Practices

✔ Keep interfaces clean.

✔ Use consistent colors.

✔ Use TextMeshPro.

✔ Minimize unnecessary animations.

✔ Group related UI elements.

✔ Test on multiple resolutions.

---

# Common Mistakes

❌ Overcrowded interfaces.

❌ Tiny text.

❌ Ignoring accessibility.

❌ Using legacy Text instead of TextMeshPro.

❌ Poor UI hierarchy.
